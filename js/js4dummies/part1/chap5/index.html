Exprimez-vous

Une expression est un morceau de code qui fournit une valeur. Les expressions peuvent servir à affecter une valeur à une variable, ou simplement avoir une valeur. Par exemple, les deux expressions suivantes sont parfaitement valides :

1 + 1
a = 1;

Les expressions peuvent être très courtes et très simples, comme dans l'exemple ci-dessus, ou être extrêmement complexes. Les morceaux de données qui servent à former une expression (comme le chiffre 1 ou la lettre a dans l'exemple précédent) sont appelés des opérandes.

Bonjour, opérateur

Un opérateur, c'est ce qui permet aux expressions de faire leur travail. Comme leur nom l'indique, les opérateurs opèrent sur les données pour fournir différents résultats. Dans les exemples ci-dessus, les signes + ou = sont des opérateurs.

Priorité des opérateurs

Une même expression contient bien souvent plusieurs opérateurs. Prenons un exemple :

a = 1 + 2 * 3 / 4;

Selon l'ordre dans lequel les calculs sont effectués, la valeur finale pourrait être l'une des trois suivantes :

a = 1.75
a = 2.5
a = 2.25

Et la bonne réponse est ... 2.5 ! L'ordre de priorité est le même qu'en maths !

L'opérateur qui porte le plus petit numéro est dit avoir la plus haute priorité. Cela peut sembler étrange, mais si vous vous figurez être la première personne d'une file d'attente (celle qui a le rang 0 en l'occurrence), et donc que vous allez être le premier à recevoir votre sandwich ou café, vous comprendrez mieux ce qui se passe.

Lorsqu'une expression contient deux opérateurs, ou plus, ayant la même priorité, ceux-ci sont évalués selon leur associativité. Cette associativité détermine si les opérateurs sont évalués en allant de la gauche vers la droite, ou de la droite vers la gauche.

Utiliser des parenthèses

Les parenthèses représentent l'opérateur qui a le plus haut niveau de priorité. Dans la plupart des cas, vous pouvez ignorer les règles décrites dans le tableau ci-dessus en regroupant simplement les opérations à l'aide de parenthèses. Par exemple, l'expression proposée vers le début (a + 1 + 2 * 3 / 4); pourrait être clarifiée de différentes manières

a = (1 + 2) * (3 / 4); // résultat : 2.25
a = (1 + (2 * 3)) / 4); // résultat : 1.75
a = ((1 + 2) * 3) / 4; // résultat : 2.25
a = 1 + ((2 * 3) / 4); // résultat : 2.5

Types d'opérateurs

Opérateurs d'affectation

Un opérateur d'affectation, comme son nom l'indique, affecte la valeur de l'opérande placé à droite de gauche. Exemple :

a = 5;

Une fois cette expression exécutée, la variable a possède comme valeur 5. Mais vous pouvez également enchaîner plusieurs opérateurs d'affectation afin de définir un contenu identique pour plusieurs variables dans une même instruction, comme dans cet exemple :

a = b = c = 5;

Du fait que l'associativité de cet opérateur s'établit de la droite vers la gauche, 5 est d'abord assigné à la variable c, puis la valeur de c est recopiée dans b, et enfin le contenu de b est affecté à a. Les trois variables a, b et enfin le contenu de b est affecté à a. Les trois variables a, b et c possèdent toutes la même valeur, 5.

Essayez maintenant de deviner la valeur finale de a une fois ces expressions évaluées :

var b = 1;
var a = b += c = 5;

Si vous ne trouvez pas la réponse, ouvrez la console JavaScript et testez par vous-même !

Opérateurs de comparaison

Les opérateurs de comparaison testent l'égalité, ou la non-égalité, entre les opérandes. Ils retournent ensuite une valeur true (vrai) ou false (faux).

Opérateur de chaîne

L'opérateur chaîne effectue des opérations en utilisant deux chaînes de caractères. Lorsqu'il est utilisé avec des chaînes, l'opérateur + devient l'opérateur de concaténation. Il sert alors à fusionner deux chaînes en une seule, en les mettant bout à bout. Notez que, dans ce cas, aucune espace n'est ajoutée. Il est donc très courant de voir des instructions comme celle qui suit, où des chaînes ne contenant rien d'autre qu'une espace blanche sont concaténées avec d'autres de manière à effectuer une séparation pour que la phrase soit cohérente :

var salut = "Bonjour, " + Prenom + ". Je suis" + " " + Humeur + " de vous voir.";

Opérateurs au niveau du bit

Les opérateurs au niveau du bit traitent les opérandes comme des représentations binaires signées sur 32 bits dans le format dit de complément à 2.

Les opérateurs au niveau du bit de JavaScript
Opérateur binaire Utilisation Description
AND a & b Renvoie un 1 pour chaque position où les bits correspondants des deux opérandes sont égaux à 1
OR a | b Renvoie un 1 pour chaque position où les bits correspondants de l'un au moins des deux opérandes sont égaux à 1
XOR a ^ b Renvoie un 1 pour chaque position où les bits correspondants de l'un des deux opérandes, mais pas des deux, sont égaux à 1
NOT ~a Inverse les bits de son opérande
Décalage à gauche a << b Décale a en représentation binaire de b (< 32) bits vers la gauche, les bits perdus étant éliminés
Décalage à droite avec propagation de signe a >> b Décale a en représentation binaire de b (< 32) bits vers la droite, les bits perdus étant éliminés
Décalage à droite avec remplissage par des 0 a >>> b Décale a en représentation binaire de b (< 32) bits vers la droite, les bits perdus sur la gauche étant remplacés par des 0

Opérateurs logiques

Les opérateurs logiques évaluent une expression (également logique !) comme étant vraie ou fausse. Il en existe trois, qui sont décrits dans le tableau ci-dessous :

Les opérateurs logiques
Opérateur Signification Description
&& || Renvoie true (vrai) si les deux opérandes sont true. Sinon, retourne false.
|| OR (ou) Renvoie false (faux) si les deux opérandes sont false. Sinon, renvoie true.
! Not (non) Inverse la valeur logique d'une unique opérande.

Opérateurs spéciaux

Opérateur conditionnel

L'opérateur conditonnel (également appelé opérateur ternaire) utilise trois opérandes. Il évalue une valeur logique, puis renvoie une valeur basée sur le résultat vrai ou faux de l'expression. Il s'agit du seul opérateur qui nécessite trois opérandes. Par exemple :

var estPlusGrandQueDix = (valeur > 10) ? "plus grand que 10" : "pas plus grand que 10";

Opérateur virgule

L'opérateur virgule évalue deux opérandes et renvoie la valeur du second. Il est le plus souvent utilisé pour réaliser des affectations multiples, ou encore d'autres opérations à l'intérieur de boucles. Il peut également servir de raccourci pour initialiser des variables. Par exemple :

var a = 10 , b = 0;

Du fait que la virgule a le plus bas niveau de priorité parmi tous les opérateurs, ses opérateurs sont toujours évalués séparément.

Opérateur de suppression

L'opérateur de suppression (delete) retire une propriété d'un objet, ou un élément d'un tableau.

Lorsque vous utilisez cet opérateur pour supprimer un élément d'un tableau, la longueur de ce tableau ne change pas. L'élément qui a été enlevé prendra simplement la valeur undefined (non défini).

var cryptos = ["BAT", "Zcash", "Neo", "EOS", "NEM"];
console.log(cryptos[4]); // renvoie "NEM";
delete cryptos[4];
console.log(cryptos[4]); // renvoie "undefined"

Opérateur in

L'opérateur in (dans) renvoie vrai (true) si la valeur spécifiée existe dans un tableau ou dans un objet.

var cryptos = ["BAT", "Zcash", "Neo", "EOS", "NEM"];
if (4 in cryptos) { console.log("Il est bien là !"); }

Opérateur instanceof

L'opérateur instanceof renvoie true si l'objet que vous spécifiez est bien du type demandé.

var maChaine = new String();
if (maChaine instanceof String) {
console.log("Mais oui, c'est une chaîne !");
}

Opérateur new

L'opérateur new crée une instance d'un objet.

var today = new Date();
var bird = new Animal();
var marguerite = new Fleur();

Opérateur this

L'opérateur this fait référence à l'objet courant. Il est fréquemment utilisé pour retrouver les propriétés d'un objet sans avoir à saisir le nom de celui-ci.

Opérateur typeof

L'opérateur typeof renvoie une chaîne contenant le type de l'opérande

var maPetiteEntreprise = "ne connaît pas la crise";
console.log(typeof maPetiteEntreprise); // renvoie "string"

Opérateur void

L'opérateur void (vide) évalue l'expression dans l'opérande sans retourner de valeur. L'usage le plus fréquent de cet opérateur se voit dans des documents HTML lorsqu'un lien est nécessaire, mais que le créateur de ce lien veut en annuler le comportement par défaut via JavaScript.

C'est un lien, mais il ne fait rien
